# Bonita AI - Data Storage Architecture

## User Data Storage Location

### Database: PostgreSQL (Neon Serverless)
All user data is stored in a PostgreSQL database hosted on Neon's serverless platform. This provides:

- **Reliability**: Enterprise-grade database with automatic backups
- **Scalability**: Automatically scales with user growth
- **Security**: Encrypted connections and data at rest
- **Performance**: Optimized for web applications

### Data Tables Structure

#### 1. User Accounts (`users` table)
- **Personal Information**: Username, email, password hash
- **Preferences**: Language, theme, color scheme, tone mode
- **Gamification**: Points, level, streak, activity stats
- **Authentication**: Secure password hashing with bcrypt

#### 2. Chat Messages (`chat_messages` table)
- **Content**: All conversations with Bonita
- **Metadata**: Language, tone mode, timestamps
- **User Isolation**: Each user's chats are completely separate

#### 3. Generated Images (`generated_images` table)
- **Image URLs**: Links to DALL-E generated images (hosted by OpenAI)
- **Prompts**: Original text prompts used for generation
- **Metadata**: Language, creation timestamps

#### 4. Video Scripts (`video_scripts` table)
- **Scripts**: Complete video scripts generated by Bonita
- **Platforms**: Target social media platforms (TikTok, Instagram, etc.)
- **Topics**: Original topics and themes

#### 5. Achievements (`achievements` table)
- **Progress Tracking**: User milestones and accomplishments
- **Points System**: Gamification rewards and levels
- **Unlock Dates**: When achievements were earned

#### 6. Analytics & Moderation
- **Content Flags**: Moderation system logs
- **Rate Limits**: Usage tracking for abuse prevention
- **Support Tickets**: User feedback and issues

### Data Isolation & Privacy

#### Per-User Data Separation
- Every data record is tied to a specific user ID
- Users can only access their own data
- No cross-user data leakage possible

#### Image Storage Details
- **Image Files**: Hosted by OpenAI (DALL-E service)
- **URLs**: Temporary signed URLs (expire after 2 hours)
- **Metadata**: Stored permanently in your database
- **Access**: Only the user who generated an image can see it

#### Authentication & Security
- **Sessions**: Secure server-side session management
- **Passwords**: Bcrypt hashing (industry standard)
- **Rate Limiting**: Prevents abuse and ensures fair usage
- **Content Moderation**: OpenAI filtering for inappropriate content

### Data Export & Portability

Users can export all their data in multiple formats:
- **JSON**: Complete structured data with all metadata
- **CSV**: Spreadsheet-compatible for analysis
- **TXT**: Human-readable format for easy review

Export includes:
- Complete chat history
- All generated images (URLs and prompts)
- Video scripts and topics
- Achievement progress
- Account statistics

### Data Retention

- **Chat Messages**: Stored indefinitely (user can clear)
- **Generated Images**: URLs expire but metadata persists
- **User Accounts**: Active until user requests deletion
- **Analytics**: Anonymized aggregated data for service improvement

### Backup & Recovery

- **Automatic Backups**: Neon provides point-in-time recovery
- **Geographic Redundancy**: Data replicated across regions
- **Export Functionality**: Users can backup their own data anytime

This architecture ensures your user data is secure, private, and always accessible while maintaining high performance and reliability for your soft launch.
