# Bonita AI - Replit Application Guide

## Overview

Bonita AI is a full-stack web application featuring a multilingual (English/Spanish/Portuguese/French) chat interface with AI-powered chat, image generation, and video script creation capabilities. The application presents itself as a "Digital Bronx Auntie" providing advice and creative assistance with a distinctive personality and cultural voice.

**Current Status**: Core functionality operational (chat, DALL-E 3 image generation, video scripts). System is stable and ready for additional enhancements.

## System Architecture

### Frontend Architecture
- **Framework**: React 18 with TypeScript
- **Build Tool**: Vite for development and production builds
- **UI Framework**: shadcn/ui components built on Radix UI primitives
- **Styling**: TailwindCSS with custom CSS variables for theming
- **Routing**: Wouter for lightweight client-side routing
- **State Management**: TanStack Query (React Query) for server state
- **Form Handling**: React Hook Form with Zod validation

### Backend Architecture
- **Runtime**: Node.js with Express.js server
- **Language**: TypeScript with ES modules
- **API Design**: RESTful endpoints for chat, image generation, and video scripts
- **AI Integration**: OpenAI API (GPT-4o model) for all AI-powered features
- **Request Validation**: Zod schemas for type-safe API validation

### Database Layer
- **ORM**: Drizzle ORM configured for PostgreSQL
- **Schema**: Defined in shared directory for type safety across frontend/backend
- **Tables**: Users and chat messages with multilingual support
- **Migrations**: Drizzle Kit for schema management

## Key Components

### Language System
- **Context Provider**: React Context for language state management
- **Persistence**: LocalStorage for language preference retention
- **Translation System**: JSON-based translations with type-safe keys
- **Toggle Component**: Custom switch component for EN/ES language switching

### Chat Interface
- **Real-time Messaging**: Stateful message history with user/AI distinction
- **Personality System**: Context-aware AI responses with cultural authenticity
- **Language Support**: Bilingual responses based on user preference
- **Message Management**: Automatic scrolling and timestamp tracking

### Image Generation
- **AI Integration**: OpenAI DALL-E 3 API for image creation
- **Fallback System**: Placeholder images when API fails
- **User Interface**: Text prompt input with generation status feedback
- **Display System**: Image preview with responsive design

### Video Script Generation
- **AI-Powered Writing**: GPT-4o optimized for script creation
- **Content Focus**: Social media and creator-focused script generation
- **Bilingual Support**: Scripts generated in user's preferred language
- **Copy Functionality**: One-click script copying to clipboard

## Data Flow

### Request Processing
1. User input captured through React components
2. Form validation using Zod schemas
3. API requests sent via TanStack Query mutations
4. Express routes validate and process requests
5. OpenAI API calls made with appropriate prompts
6. Responses formatted and returned to client
7. UI updates with loading states and results

### State Management
- **Server State**: TanStack Query handles API data caching and synchronization
- **Client State**: React useState for component-local state
- **Global State**: React Context for language preferences
- **Persistence**: LocalStorage for user preferences

## External Dependencies

### Core Runtime
- **Node.js 20**: JavaScript runtime environment
- **Express.js**: Web application framework
- **TypeScript**: Type-safe JavaScript development

### AI Services
- **OpenAI API**: GPT-4o for chat completions and DALL-E 3 for image generation
- **API Key**: Required environment variable (OPENAI_API_KEY)

### UI Libraries
- **Radix UI**: Accessible component primitives
- **TailwindCSS**: Utility-first CSS framework
- **Lucide React**: Icon library for UI elements

### Database
- **PostgreSQL**: Primary database (configured but not actively used in current implementation)
- **Neon Database**: Serverless PostgreSQL provider
- **Drizzle ORM**: Type-safe database operations

## Deployment Strategy

### Development Environment
- **Replit Integration**: Configured for Replit development environment
- **Hot Reload**: Vite dev server with HMR for frontend changes
- **Process Management**: Single command (`npm run dev`) starts both frontend and backend
- **Port Configuration**: Frontend proxies API requests to backend on port 5000

### Production Build
- **Build Process**: Vite builds frontend static assets
- **Server Bundle**: esbuild creates optimized backend bundle
- **Asset Serving**: Express serves static files in production
- **Environment Variables**: Database URL and OpenAI API key required

### Replit Configuration
- **Modules**: Node.js 20, Web, and PostgreSQL 16 enabled
- **Deployment**: Autoscale deployment target
- **Port Mapping**: Internal port 5000 mapped to external port 80
- **Workflows**: Automated startup and development workflows
